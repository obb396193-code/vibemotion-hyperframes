# 节奏与分镜方法论（提炼自官方 launches `STORYBOARD.md`）

> 用途：做片**前**先按这套定故事骨架 + 分镜 + 节奏，再去填 CHECKPOINT 的分场清单。这是官方 HeyGen 团队 `STORYBOARD.md` 的打法，直接喂养 vibemotion 的脚本/分镜环节。

## 1. 先定故事骨架（archetype）
一句话能说清的叙事脊。官方常用 **Before–After–Bridge**：痛点 → 转折 → 解法。
- **钩子 = 痛点验证**（不是夸产品）。例：本地渲染卡在 1%、要等 15 分钟——先让观众"疼"。
- 收尾 = 闭合钩子（回扣开头那个梗）。

## 2. 场要短、要密
MG 短视频**每场 2–3.5 秒**，30s ≈ 8 场。给每场标 `起–止 + 时长 + 这场讲什么`。别一场拖到 8 秒。

## 3. 节拍菜单（一条片由这些 beat 拼）
钩子(痛点) → **连接节拍**(1–1.5s 的 thinking/checking 小停顿) → **证据**(真实日志/数据，高亮逐行扫) → **文字节拍**(单行金句) → 解法 → **技术展示**(快、点到为止) → payoff(闭合钩子) → logo/CTA。

## 4. 转场要省（关键）
全片只在 **1–3 处叙事跃迁**用 shader/大转场（如转折处、payoff 处）；其余用**硬切 / morph**。
> 「每个切都加转场 = 廉价」。官方 30s 片只有 **2 处** camera seam：转折用 leftward cut-the-curve、payoff 用 zoom-through。

## 5. 连续性装置（串起全片）
找**一个持续元素**贯穿始终。官方=**同一个 Claude 窗口**，对话 history 不断累积、prompt 框始终在，**从不换新卡片**。给观众"这是一段连续的事"的感觉。

## 6. 真实感 > 假道具
用**真实代码/日志/界面**（真 ascii 渲染日志、真 CLI 命令），**绝不用假进度条、假 job 药丸**。真实=可信。

## 7. 配色纪律
底色统一（官方=暖纸底贯穿）；**强调色只在关键一帧爆一次**——官方让金句 "machine." 在收束时变成 mint→cyan 渐变，"the only color on the frame"。别处处上色。

## 8. 对比即结构
把 before/after 做成**可视对比**：官方 S3「1 个本地 worker·fps 1.6·eta 14分」直接对打 S6「16 worker 并行」。结构本身在讲点。

## 9. 微指导要具体到帧
分镜里把"看点"指到具体动作：**光标升到 ↵ 点击**、**mint 高亮逐行下扫、停在 eta 行**（"visceral proof"）。别只写"展示终端"。

## 自动化：自愈渲染循环 + 按需多角色（借鉴 vibe-video 的**思想**，不搬框架）
vibe-video 的真正价值是 **Animator 的自愈闭环**，把它固化成 SOP：
> **规划分场 → 写 HTML+GSAP → lint → render → 抽帧自检 → 有问题就照「症状→病因→修法」表改了再渲 → 循环到干净且好看。**

**多角色**（研究/读码/动画）我们**用 Claude Code 原生 subagent 按需起**即可，不需要 agno 框架：
- 选题要联网研究 → 起一个 research subagent；
- 要"讲解某 repo" → 起一个 code-explorer subagent（clone 只读）；
- 复杂片渲染 → 起一个 animator subagent，让它跑上面的自愈循环。
> ⚠️ **别每条片都套全套多 agent**——简单片自己直接做更快。角色是"按需调用"，不是"固定流水线"。
